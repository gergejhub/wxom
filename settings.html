This build is zero‑manual configuration. Runway headings are sourced automatically (OurAirports) during dataset updates.

Data sources

  • METAR/TAF: aviationweather.gov (server-side fetch via GitHub Actions)
  • Runway geometry (heading/width): OurAirports runways.csv (server-side; used for crosswind advisory)

Operational decisions remain the responsibility of the flight crew/dispatcher. This tool provides advisory flags based on OM extracts and available data.

About OM policy layer

Some OM rules require inputs not available in METAR/TAF (e.g., RWYCC / SNOWTAM, selected runway). Where data is missing, the tool applies conservative heuristics or marks values as N/A.
